Asistente Web IA - Gestión de Incidentes is a full-stack cybersecurity tool that lets employees submit suspicious files, URLs, and screenshots for automated analysis against VirusTotal, MetaDefender Cloud, Google Safe Browsing, and Google Gemini. This page walks you through every step needed to get the application running and perform your first analysis.

## Prerequisites

Before you begin, make sure you have the following:

- **Python 3.11** or higher
- **Node.js 18** or higher
- **Docker** and **Docker Compose** (only required for the Docker path)
- API keys for VirusTotal, MetaDefender Cloud, Google Safe Browsing, and Google Gemini

<Tip>
  All four security APIs operate within their free tiers. You can obtain keys from the VirusTotal developer portal, the MetaDefender Cloud dashboard, and the Google Cloud Console (for both Safe Browsing and Gemini).
</Tip>

## Installation

<Tabs>
  <Tab title="Without Docker">
    Run the backend and frontend as separate local processes. Best for active development.

    <Steps>
      <Step title="Clone the repository">
        ```bash
        git clone https://github.com/ryan-alej19/Asistente-Web-IA-Gestion-Incidentes-Luis-Mera.git
        cd Asistente-Web-IA-Gestion-Incidentes-Luis-Mera
        ```
      </Step>

      <Step title="Create a Python virtual environment">
        ```bash
        cd backend
        python -m venv venv
        ```

        Activate the environment for your operating system:

        <CodeGroup>
          ```bash Windows
          venv\Scripts\activate
          ```

          ```bash Linux / macOS
          source venv/bin/activate
          ```
        </CodeGroup>
      </Step>

      <Step title="Install Python dependencies">
        ```bash
        pip install -r requirements.txt
        ```

        Key packages installed include Django 5.1.4, Django REST Framework, `djangorestframework-simplejwt`, `google-generativeai`, `gunicorn`, and `whitenoise`.
      </Step>

      <Step title="Configure environment variables">
        ```bash
        cp .env.example .env
        ```

        Open `backend/.env` and fill in your API keys:

        ```env backend/.env
        SECRET_KEY=your-secret-key-here
        DEBUG=True
        CORS_ALLOWED_ORIGINS=http://localhost:3000

        VIRUSTOTAL_API_KEY=your-key
        METADEFENDER_API_KEY=your-key
        GOOGLE_API_KEY=your-key
        GOOGLE_SAFE_BROWSING_KEY=your-key
        ```

        <Warning>
          Never commit `.env` to version control. It is listed in `.gitignore` by default — keep it that way.
        </Warning>
      </Step>

      <Step title="Run database migrations and start the backend">
        ```bash
        python manage.py migrate
        python manage.py runserver
        ```

        The Django API is now available at `http://localhost:8000`.
      </Step>

      <Step title="Install frontend dependencies and start the dev server">
        Open a **new terminal** in the project root and run:

        ```bash
        cd frontend
        npm install
        npm start
        ```

        The React app opens at `http://localhost:3000`.
      </Step>
    </Steps>
  </Tab>

  <Tab title="Docker Compose">
    Run both services as containers with a single command. Best for a self-contained local environment.

    <Steps>
      <Step title="Clone the repository">
        ```bash
        git clone https://github.com/ryan-alej19/Asistente-Web-IA-Gestion-Incidentes-Luis-Mera.git
        cd Asistente-Web-IA-Gestion-Incidentes-Luis-Mera
        ```
      </Step>

      <Step title="Configure environment variables">
        ```bash
        cp backend/.env.example backend/.env
        ```

        Edit `backend/.env` with your API keys (same values as the local path above).

        <Warning>
          Never commit `.env` to version control. It is listed in `.gitignore` by default — keep it that way.
        </Warning>
      </Step>

      <Step title="Build and start the containers">
        ```bash
        docker-compose up --build -d
        ```

        Docker Compose starts two containers:

        | Container | Base image | Exposed port |
        | --- | --- | --- |
        | `soc_backend` | `python:3.11-slim` | `8000` |
        | `soc_frontend` | `node:18-slim` | `3000` |

        The frontend container automatically receives `REACT_APP_API_URL=http://localhost:8000` so the React app points to the local Django server.
      </Step>

      <Step title="Verify the containers are running">
        ```bash
        docker-compose ps
        ```

        Both services should report a status of `Up`.
      </Step>
    </Steps>

    To stop the application:

    ```bash
    docker-compose down
    ```
  </Tab>
</Tabs>

## Log in for the first time

Open `http://localhost:3000` in your browser and sign in with one of the pre-seeded test accounts:

| Username | Password | Role | Access level |
| --- | --- | --- | --- |
| `admin` | `admin123` | Administrator | Manage users, incidents, and analyses |
| `analista` | `analista123` | Analyst | Review incidents, update statuses, view dashboard |
| `empleado` | `empleado123` | Employee | Submit files/URLs for analysis, create incident reports |

<Note>
  These credentials are for local development and testing only. Change all passwords before exposing the application on a network.
</Note>

## Run your first analysis

1. Log in as `empleado` / `empleado123`.
2. Navigate to the **Análisis** section from the sidebar.
3. Upload a suspicious file or paste a URL into the input field.
4. Wait 8–12 seconds while the engine queries VirusTotal, MetaDefender, Google Safe Browsing, and Gemini in parallel.
5. Review the color-coded risk indicator (red, yellow, blue, or green) and the AI-generated plain-language explanation.
6. If the result looks dangerous, click **Crear Reporte** to open an incident ticket.

<Tip>
  Repeated analyses of the same file or URL resolve in under 500 ms thanks to the built-in response cache, which reduces external API calls by approximately 65%.
</Tip>
