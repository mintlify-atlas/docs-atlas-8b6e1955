---
title: "User management API for administrators"
description: "Reference for the admin-only user management endpoints in Asistente Web IA — list accounts, toggle active status, change roles, and audit login attempts."
---

The **Asistente Web IA - Gestión de Incidentes** user management endpoints are restricted exclusively to users with the `admin` role. They allow administrators to view all accounts, activate or deactivate users, assign roles, and review the login-attempt audit log. All endpoints require a valid token in the `Authorization` header.

<Warning>
  Attempting to deactivate your own account via the toggle-status endpoint is blocked by the server with a `400` error. Attempting to change your own role is also blocked. These restrictions are enforced regardless of your admin status.
</Warning>

---

## GET /api/users/list/

Returns all registered user accounts ordered by most recently joined.

### Response

Array of user objects.

<ResponseField name="id" type="integer">
  Django user primary key.
</ResponseField>

<ResponseField name="username" type="string">
  Login username.
</ResponseField>

<ResponseField name="email" type="string">
  User email address.
</ResponseField>

<ResponseField name="is_active" type="boolean">
  Whether the account can authenticate.
</ResponseField>

<ResponseField name="role" type="string">
  Role from the user's `UserProfile`. One of `employee`, `analyst`, or `admin`. Returns `employee` if no profile exists.
</ResponseField>

<ResponseField name="last_login" type="string">
  ISO 8601 timestamp of the last successful login, or `null`.
</ResponseField>

<ResponseField name="last_login_formatted" type="string">
  Human-readable last login date as `YYYY-MM-DD HH:MM`, or `"Nunca"` if the user has never logged in.
</ResponseField>

<ResponseField name="date_joined" type="string">
  ISO 8601 timestamp of account creation.
</ResponseField>

<ResponseField name="date_joined_formatted" type="string">
  Human-readable join date as `YYYY-MM-DD`.
</ResponseField>

```bash curl
curl --request GET \
  --url https://your-backend.onrender.com/api/users/list/ \
  --header 'Authorization: Token YOUR_TOKEN'
```

## PATCH /api/users/\<id\>/toggle\_status/

Toggles the `is_active` flag of a user account. If the account is active it is deactivated, and vice versa. A deactivated user cannot obtain a token or access any protected endpoint.

### Path parameters

<ParamField path="id" type="integer" required>
  The Django user ID to toggle.
</ParamField>

### Response

```json
{
  "message": "Usuario johndoe desactivado correctamente",
  "is_active": false
}
```

```bash curl
curl --request PATCH \
  --url https://your-backend.onrender.com/api/users/5/toggle_status/ \
  --header 'Authorization: Token YOUR_TOKEN'
```

## PATCH /api/users/\<id\>/change\_role/

Updates the `role` field on a user's `UserProfile`. Creates the profile if it does not yet exist.

### Path parameters

<ParamField path="id" type="integer" required>
  The Django user ID whose role to change.
</ParamField>

### Request body

<ParamField body="role" type="string" required>
  New role to assign. One of `employee`, `analyst`, or `admin`.
</ParamField>

### Response

```json
{
  "message": "Rol de johndoe actualizado a analyst",
  "role": "analyst"
}
```

```bash curl
curl --request PATCH \
  --url https://your-backend.onrender.com/api/users/5/change_role/ \
  --header 'Authorization: Token YOUR_TOKEN' \
  --header 'Content-Type: application/json' \
  --data '{"role": "analyst"}'
```

## GET /api/users/login-attempts/

Returns the full login-attempt audit log, ordered by most recent first. This log records both successful and failed login attempts including the client IP address and user agent.

The endpoint is paginated with a default page size of 50 and a maximum of 1000.

### Response

Array of login-attempt objects.

<ResponseField name="id" type="integer">
  Auto-incremented record ID.
</ResponseField>

<ResponseField name="username" type="string">
  Username string that was submitted in the login request. May not correspond to an existing account if the attempt failed.
</ResponseField>

<ResponseField name="ip_address" type="string">
  Source IP address of the request, or `null` if not available.
</ResponseField>

<ResponseField name="user_agent" type="string">
  `User-Agent` header from the request, or `null`.
</ResponseField>

<ResponseField name="successful" type="boolean">
  `true` if the login succeeded; `false` if credentials were invalid.
</ResponseField>

<ResponseField name="timestamp" type="string">
  ISO 8601 timestamp of the attempt.
</ResponseField>

<ResponseField name="timestamp_formatted" type="string">
  Human-readable timestamp as `YYYY-MM-DD HH:MM:SS`.
</ResponseField>

```bash curl
curl --request GET \
  --url 'https://your-backend.onrender.com/api/users/login-attempts/?page=1' \
  --header 'Authorization: Token YOUR_TOKEN'
```
