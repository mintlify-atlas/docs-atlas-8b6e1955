The **Asistente Web IA - Gestión de Incidentes** API exposes a set of endpoints for creating, querying, and managing security incidents. An incident represents a potentially threatening artifact — a URL, file, or image — that has been submitted for automated analysis. Employees can report incidents and view their own submissions; analysts and administrators have full visibility across all incidents and can update their status, add notes, and export reports.

## Incident data model

Each incident is stored in the `Incident` table with the following fields.

### Core fields

<ResponseField name="id" type="integer">
  Auto-incremented primary key.
</ResponseField>

<ResponseField name="incident_type" type="string" required>
  Category of the submitted artifact. One of `url`, `file`, or `image`.
</ResponseField>

<ResponseField name="url" type="string">
  The target URL when `incident_type` is `url`. Nullable.
</ResponseField>

<ResponseField name="attached_file" type="string">
  Relative path to the uploaded file stored under `incidents/`. Nullable.
</ResponseField>

<ResponseField name="description" type="string">
  Free-text description provided by the reporter at submission time. Defaults to an empty string.
</ResponseField>

### Analysis fields

<ResponseField name="risk_level" type="string">
  Computed risk classification. One of `LOW`, `MEDIUM`, `HIGH`, `CRITICAL`, or `UNKNOWN`. Defaults to `UNKNOWN`.
</ResponseField>

<ResponseField name="virustotal_result" type="object">
  JSON snapshot of the VirusTotal engine result mapped from the `analysis_result` payload. Nullable.
</ResponseField>

<ResponseField name="metadefender_result" type="object">
  JSON snapshot of the MetaDefender engine result. Nullable.
</ResponseField>

<ResponseField name="gemini_analysis" type="string">
  Plain-text AI explanation and recommendation generated by Google Gemini. Stored as `{explanation}\n\nRecomendación: {recommendation}`.
</ResponseField>

<ResponseField name="analysis_result" type="object">
  Full analysis snapshot as received from the preview endpoints. Contains `engines`, `risk_level`, `positives`, `total`, `gemini_explicacion`, and `gemini_recomendacion`. This is the primary source of truth used by the PDF and detail views.
</ResponseField>

### Status and control fields

<ResponseField name="status" type="string">
  Workflow state. One of `pending`, `investigating`, or `resolved`. Defaults to `pending`.
</ResponseField>

<ResponseField name="analyst_notes" type="string">
  Legacy free-text notes field on the incident record. New notes are created as `IncidentNote` objects via the notes endpoint.
</ResponseField>

<ResponseField name="reported_by" type="integer">
  Foreign key to the Django `auth.User` who submitted the incident.
</ResponseField>

<ResponseField name="created_at" type="string">
  ISO 8601 timestamp. Set automatically on creation.
</ResponseField>

<ResponseField name="updated_at" type="string">
  ISO 8601 timestamp. Updated automatically on every save.
</ResponseField>

## Role-based access

| Role | Incident visibility |
|---|---|
| `employee` | Only incidents where `reported_by` matches the authenticated user |
| `analyst` | All incidents across all users |
| `admin` | All incidents across all users |

Employees who request an incident they did not report receive a `403` response.

## Available endpoints

| Method | Path | Description | Auth |
|---|---|---|---|
| `POST` | `/api/incidents/create/` | Submit a new incident | Any authenticated user |
| `GET` | `/api/incidents/list/` | List incidents with filters and pagination | Any authenticated user |
| `GET` | `/api/incidents/<id>/` | Retrieve a single incident | Any authenticated user |
| `PATCH` | `/api/incidents/<id>/update-status/` | Update status and add a note | Analyst / Admin |
| `GET` | `/api/incidents/<id>/notes/` | List internal notes on an incident | Analyst / Admin |
| `POST` | `/api/incidents/<id>/notes/` | Add an internal note | Analyst / Admin |
| `GET` | `/api/incidents/stats/` | Dashboard aggregated statistics | Analyst / Admin |
| `GET` | `/api/incidents/<id>/pdf/` | Download a single-incident PDF report | Any authenticated user |
| `GET` | `/api/incidents/report/pdf/monthly/` | Download a monthly PDF report | Analyst / Admin |
| `GET` | `/api/incidents/export/csv/` | Download all incidents as CSV | Analyst / Admin |
| `GET` | `/api/health/` | Backend health check | None |
